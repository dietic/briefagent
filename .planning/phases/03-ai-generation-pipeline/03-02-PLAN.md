    1. Create `src/routes/api/generate/plan/+server.ts` (POST endpoint):
       - Auth check: `const user = await locals.getUser(); if (!user) return error(401, 'Unauthorized');`
       - Parse request body: `{ productId }` (validate productId is a string)
       - Verify product belongs to user: query products table where id = productId AND userId = user.id
       - If not found, return 404
       - Create job record in generationJobs table:
         ```
         const [job] = await db.insert(generationJobs).values({
           userId: user.id,
           productId,
           type: 'content_plan',
           status: 'pending',
           totalSteps: 3,
         }).returning();
         ```
       - Fire-and-forget the actual generation (do NOT await in the request handler):
         ```
         runPlanGeneration(job.id, user.id, productId).catch(err => {
           console.error('Plan generation failed:', err);
         });
         ```
       - Return `json({ jobId: job.id })` immediately

       Implement `async function runPlanGeneration(jobId, userId, productId)`:
       - Step 1: Update job status to 'running', currentStep 1, progress 'Assembling brief...'
         Call `assembleBrief(productId)`
       - Step 2: Update currentStep 2, progress 'Generating content plan...'
         Query previous plan summaries: `db.query.contentPlans.findMany({ where: eq(contentPlans.productId, productId), columns: { summary: true, contentThemes: true } })`
         Call `generateContentPlan(brief, summaries)`
       - Step 3: Update currentStep 3, progress 'Saving content plan...'
         Insert into contentPlans table (strategyOverview, contentThemes, productId)
         Auto-generate a summary: first 200 chars of strategyOverview + themes joined
         Insert all post slots into posts table (map each slot to a row with contentPlanId, productId, scheduledAt parsed from dateTime, etc.)
         Update job: resultId = contentPlan.id, status 'completed', progress 'Done!'
       - Catch: Update job status 'failed', error message

       Helper `async function updateJob(jobId, updates)`: `db.update(generationJobs).set({ ...updates, updatedAt: new Date() }).where(eq(generationJobs.id, jobId))`

    2. Create `src/routes/api/jobs/[id]/stream/+server.ts` (GET, SSE endpoint):
       - Auth check via locals.getUser()
       - Verify job belongs to user (query generationJobs where id = params.id)
       - If job not found or userId mismatch, return 404
       - Return a ReadableStream with SSE format:
         - Poll the generationJobs table every 1000ms
         - Send `data: {"status":"running","progress":"...","currentStep":2,"totalSteps":3}\n\n`
         - When status is 'completed' or 'failed', send final event and close stream
         - Handle stream cancellation (set closed flag)
       - Headers: Content-Type text/event-stream, Cache-Control no-cache, Connection keep-alive

    3. Create `src/routes/api/jobs/[id]/+server.ts` (GET, polling fallback):
       - Auth check
       - Query generationJobs where id = params.id and userId = user.id
       - If not found, return 404
       - Return json with job status, progress, currentStep, totalSteps, error, resultId
  </action>

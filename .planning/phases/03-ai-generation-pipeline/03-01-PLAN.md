  <name>Task 1: Install AI SDK and extend DB schema with generation pipeline tables</name>
  <files>

  <action>
    1. Install AI SDK packages:
       ```
       pnpm add ai @ai-sdk/openai
       ```

    2. Extend `src/lib/server/db/schema.ts` with new enums and tables. Keep ALL existing tables (products, productBriefs, assets) untouched. Add AFTER existing code:

       **Enums (using pgEnum):**
       - `jobStatusEnum`: 'pending', 'running', 'completed', 'failed'
       - `jobTypeEnum`: 'content_plan', 'post_generation'
       - `contentCategoryEnum`: 'educational', 'promotional', 'social_proof', 'behind_the_scenes', 'engagement', 'tips', 'announcement', 'storytelling'
       - `postStatusEnum`: 'draft', 'pending_review', 'approved', 'rejected', 'scheduled'

       **Tables:**

       `generationJobs` table:
       - id: uuid PK defaultRandom
       - userId: uuid notNull
       - productId: uuid FK -> products.id onDelete cascade
       - type: jobTypeEnum notNull
       - status: jobStatusEnum default 'pending' notNull
       - progress: text (human-readable progress message)
       - currentStep: integer default 0
       - totalSteps: integer default 1
       - error: text
       - resultId: uuid (ID of created contentPlan or null)
       - createdAt: timestamp withTimezone defaultNow
       - updatedAt: timestamp withTimezone defaultNow

       `contentPlans` table:
       - id: uuid PK defaultRandom
       - productId: uuid FK -> products.id onDelete cascade
       - strategyOverview: text notNull
       - contentThemes: text array notNull
       - summary: text (short summary for PLAN-06 context -- avoids including full plan text in future prompts)
       - createdAt: timestamp withTimezone defaultNow

       `posts` table:
       - id: uuid PK defaultRandom
       - contentPlanId: uuid FK -> contentPlans.id onDelete cascade
       - productId: uuid FK -> products.id onDelete cascade
       - platform: text notNull default 'linkedin'
       - postType: text notNull (values: 'static_image', 'text_only')
       - scheduledAt: timestamp withTimezone
       - topic: text notNull
       - contentCategory: contentCategoryEnum notNull
       - keyMessage: text notNull
       - assetReferences: text array
       - copyText: text (generated copy -- null until copy generation runs)
       - hashtags: text array (generated hashtags)
       - imageUrl: text (URL in Supabase Storage -- null until image generation runs)
       - imagePrompt: text (stored for debugging/regeneration)
       - status: postStatusEnum default 'draft' notNull
       - createdAt: timestamp withTimezone defaultNow
       - updatedAt: timestamp withTimezone defaultNow

    3. Add Drizzle `relations` for the new tables (import `relations` from 'drizzle-orm'):
       - `generationJobsRelations`: generationJobs.productId -> products (many-to-one)
       - `contentPlansRelations`: contentPlans.productId -> products (many-to-one), contentPlans -> posts (one-to-many)
       - `postsRelations`: posts.contentPlanId -> contentPlans (many-to-one), posts.productId -> products (many-to-one)
       - Also add to existing `products` table relations if not already present: products -> contentPlans (one-to-many), products -> posts (one-to-many), products -> generationJobs (one-to-many)

    4. Export all new tables and enums.

    NOTE: Do NOT add `boolean` import again if already imported. Check existing imports and extend them. The existing schema uses `pgTable, uuid, text, timestamp, boolean, jsonb, integer` from 'drizzle-orm/pg-core'. You need to add `pgEnum` to that import.
  </action>

    - `grep "relations" src/lib/server/db/schema.ts` shows relation definitions

    1. Create `src/lib/server/ai/providers.ts`:

       ```typescript
       import { openai } from '@ai-sdk/openai';

       // Text models
       export const planModel = openai('gpt-4.1');           // Strategic planning (higher quality)
       export const copyModel = openai('gpt-4.1-mini');      // Per-post copy (cost-effective)
       export const analysisModel = openai('gpt-4.1-mini');  // Brand analysis (vision-capable)

       // Image model
       export const imageModel = openai.image('gpt-image-1');
       ```
       The `@ai-sdk/openai` provider reads `OPENAI_API_KEY` from `process.env` automatically -- no explicit env var import needed.

    2. Create `src/lib/server/ai/pipeline/brief-assembler.ts`:
       - Import `db` from `$lib/server/db`
       - Import `products, productBriefs, assets` from `$lib/server/db/schema`
       - Import `eq` from `drizzle-orm`

       Define and export `AssembledBrief` interface:
       ```typescript
       export interface AssembledBrief {
         product: {
           id: string;
           name: string;
           description: string | null;
           websiteUrl: string | null;
           scrapedData: Record<string, unknown> | null;
         };
         brief: {
           problemSolved: string | null;
           keyFeatures: string[] | null;
           differentiator: string | null;
           pricingInfo: string | null;
           productStage: string | null;
           idealCustomer: string | null;
           industry: string | null;
           ageRange: string | null;
           painPoints: string[] | null;
           audienceHangouts: string[] | null;
           personalityTraits: string[] | null;
           exampleContent: string | null;
           wordsToUse: string[] | null;
           wordsToAvoid: string[] | null;
           mainGoal: string | null;
           postingFrequency: string | null;
         };
         assets: Array<{
           id: string;
           fileUrl: string;
           tag: string | null;
           description: string | null;
           isPrimary: boolean | null;
         }>;
       }
       ```

       Export `async function assembleBrief(productId: string): Promise<AssembledBrief>`:
       - Query product via `db.query.products.findFirst({ where: eq(products.id, productId) })`
       - If not found, throw Error('Product not found')
       - Query brief via `db.query.productBriefs.findFirst({ where: eq(productBriefs.productId, productId) })`
       - Query assets via `db.query.assets.findMany({ where: eq(assets.productId, productId) })`
       - Return assembled object mapping fields from DB results
       - If no brief found, return empty brief object with all null values
       - Cast `product.scrapedData` as `Record<string, unknown> | null`

    3. Update `.env.example` -- add `OPENAI_API_KEY=` after the existing Supabase vars.
  </action>

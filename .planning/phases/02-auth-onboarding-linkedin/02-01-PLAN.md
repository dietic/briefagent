    6. Update `src/hooks.server.ts` to chain Supabase SSR middleware with the existing Paraglide middleware. Use SvelteKit's `sequence` helper from `@sveltejs/kit/hooks` to compose both handles:
       - The Supabase handle creates a server client via `createServerClient` from `@supabase/ssr`, attaches it to `event.locals.supabase`, and attaches `event.locals.getUser` (which calls `supabase.auth.getUser()` -- NOT `getSession()`).
       - The Paraglide handle stays as-is for i18n.
       - Export: `export const handle = sequence(supabaseHandle, paraglideHandle);`
       - Filter serialized response headers to include `content-range` and `x-supabase-api-version`.
       Follow the exact pattern from 02-RESEARCH.md Pattern 1.

    7. Create `src/lib/server/auth.ts` with a helper:
       ```typescript
       import { redirect, error } from '@sveltejs/kit';
       import type { RequestEvent } from '@sveltejs/kit';
       export async function requireAuth(event: RequestEvent) {
         const user = await event.locals.getUser();
         if (!user) throw redirect(303, '/login');
         return user;
       }
       ```

    8. Create `src/routes/dashboard/+layout.server.ts` -- auth guard:
       ```typescript
       import { redirect } from '@sveltejs/kit';
       import type { LayoutServerLoad } from './$types';
       export const load: LayoutServerLoad = async ({ locals }) => {
         const user = await locals.getUser();
         if (!user) throw redirect(303, '/login');
         return { user };
       };
       ```

    9. Update `src/routes/dashboard/+layout.svelte` to receive the `user` data from the server load and pass it to the Sidebar (for the user pill at the bottom). Add the `data` prop:

  <action>
    **Step 1: Create the SvelteKit project**

    Use the `sv` CLI to scaffold a new SvelteKit project in the current directory:
    ```bash
    npx sv create . --template minimal --types ts --no-add-ons --no-install
    ```
    Then add integrations:
    ```bash
    npx sv add tailwindcss
    npx sv add paraglide  # Configure with languageTags: en,es and default: en
    ```

    **Step 2: Install dependencies**

    ```bash
    npm install bits-ui lucide-svelte mode-watcher svelte-inview
    npm install @fontsource/inter @fontsource-variable/plus-jakarta-sans
    ```

    **Step 3: Create the design token file**

    Create `src/lib/styles/theme.css` with the COMPLETE @theme directive from the RESEARCH.md Pattern 1. This includes:
    - Reset defaults: `--color-*: initial;` and `--font-*: initial;`
    - Primary colors: Vibrant violet-purple (OKLCH hue ~275-285), full scale 50-950
    - Accent colors: Warm orange-coral (OKLCH hue ~38-55), 400/500/600
    - Neutral colors: Clean achromatic grays, full scale 50-950
    - Surface colors: Light and dark backgrounds, muted variants
    - Semantic colors: success, warning, error, info
    - White and black
    - Typography: font-display (Plus Jakarta Sans), font-body (Inter), font-mono (JetBrains Mono)
    - Text sizes: xs through 7xl
    - Font weights: normal, medium, semibold, bold, extrabold
    - Line heights: tight, snug, normal, relaxed
    - Letter spacing: tight, normal, wide
    - Spacing: 4px base (`--spacing: 0.25rem`)
    - Border radii: sm, md, lg, xl, 2xl, full
    - Shadows: sm, md, lg, xl, glow, glow-lg (glow uses primary-500 color)
    - Animations: fade-in, fade-in-up, fade-in-down, slide-in-left, slide-in-right, scale-in, float, pulse-glow, gradient-shift -- with full @keyframes
    - Easing functions: ease-out, ease-spring, ease-smooth
    - Breakpoints: sm (40rem), md (48rem), lg (64rem), xl (80rem), 2xl (96rem)

    Use the exact OKLCH values from RESEARCH.md. These are calibrated for the Gumloop/Relume.io aesthetic.

    **Step 4: Configure app.css**

    Update `src/app.css` to:
    ```css
    @import "tailwindcss";
    @import "./lib/styles/theme.css";

    @custom-variant dark (&:where(.dark, .dark *));

    :root {
      --bg-primary: var(--color-surface-light);
      --bg-muted: var(--color-surface-muted-light);
      --text-primary: var(--color-neutral-900);
      --text-secondary: var(--color-neutral-600);
      --text-muted: var(--color-neutral-400);
      --border-default: var(--color-neutral-200);
    }

    :root:where(.dark, .dark *) {
      --bg-primary: var(--color-surface-dark);
      --bg-muted: var(--color-surface-muted-dark);
      --text-primary: var(--color-neutral-50);
      --text-secondary: var(--color-neutral-300);
      --text-muted: var(--color-neutral-500);
      --border-default: var(--color-neutral-800);
    }
    ```

    **Step 5: Configure app.html**

    Update `src/app.html`:
    - Set `<html lang="%lang%">` for Paraglide locale injection
    - Preload font woff2 files for Plus Jakarta Sans Variable and Inter 400/500/600/700 in the `<head>`
    - Ensure `%sveltekit.head%` and `%sveltekit.body%` are present

    **Step 6: Set up Paraglide hooks**

    Create `src/hooks.server.ts` with the Paraglide middleware pattern from RESEARCH.md:
    - Import `paraglideMiddleware` from `$lib/paraglide/server`
    - Transform page HTML to inject locale into `%lang%`

    Create `src/hooks.ts` with the reroute hook:
    - Import `deLocalizeUrl` from `$lib/paraglide/runtime`
    - Reroute to strip locale prefix from URL paths

    **Step 7: Create i18n message files**

    Create `messages/en.json` with ALL landing page text using the prefix convention `landing_{section}_{element}`:
    - Navigation: `nav_features`, `nav_how_it_works`, `nav_pricing`, `nav_get_started`, `nav_brand_name`
    - Hero: `landing_hero_tagline`, `landing_hero_description`, `landing_hero_cta`, `landing_hero_cta_secondary`
    - Features: `landing_features_title`, `landing_features_subtitle`, plus 6 feature titles and descriptions (AI Content Strategy, Original AI Visuals, Platform-Optimized Copy, One-Click Publishing, Deep Brand Understanding, Review and Approve)
    - How it works: `landing_how_title`, `landing_how_subtitle`, 3 steps (Brief your product, AI generates your strategy, Review and publish) -- each with title and description
    - Pricing: `landing_pricing_title`, `landing_pricing_subtitle`, `landing_pricing_beta_badge`, `landing_pricing_coming_soon`, `landing_pricing_cta`, plus tier names/prices/descriptions/features for Free/Pro/Agency
    - CTA: `landing_cta_title`, `landing_cta_description`, `landing_cta_button`
    - Footer: `footer_tagline`, `footer_product`, `footer_company`, `footer_legal`, relevant links
    - UI shared: `ui_theme_toggle`, `ui_language_label`

    Create `messages/es.json` with complete Spanish translations for every key. Spanish text is typically 15-30% longer -- ensure translations are natural, not machine-translated. Use informal "tu" form for the target audience (indie hackers, founders).

    **Step 8: Set up root layout**

    Update `src/routes/+layout.svelte`:
    - Import `../app.css`
    - Import font CSS: `@fontsource/inter/400.css`, `@fontsource/inter/500.css`, `@fontsource/inter/600.css`, `@fontsource/inter/700.css`, `@fontsource-variable/plus-jakarta-sans`
    - Import and render `ModeWatcher` from `mode-watcher`
    - Render `{@render children()}`

    Update `src/routes/+page.svelte` to a minimal placeholder that imports a Paraglide message and displays it, proving i18n works.

    **Important notes:**
    - Do NOT use shadcn-svelte or any pre-styled component library. All styling is custom via @theme tokens.
    - Do NOT hardcode any text strings. Every visible string must go through Paraglide `m.key()`.
    - Verify `npm run dev` starts without errors after setup.
    - Verify `npm run build` completes without errors.

  <action>
    Build the design system component library that the landing page and future app phases will consume. Every component uses Tailwind utility classes referencing the @theme tokens. All components support dark mode via Tailwind's `dark:` variant. All text content is passed as children/props (components themselves have no hardcoded text -- i18n happens at the usage site).

    **Button component** (`src/lib/components/ui/button.svelte`):
    - Props: `variant` ('primary' | 'secondary' | 'ghost' | 'outline'), `size` ('sm' | 'md' | 'lg'), `href` (optional, renders `<a>` instead of `<button>`), `children` (Snippet), `class` (optional override), `...rest` (spread)
    - Styling per variant:
      - primary: `bg-primary-500 text-white hover:bg-primary-600 hover:shadow-glow`
      - secondary: `bg-neutral-100 dark:bg-neutral-800 text-neutral-900 dark:text-neutral-100 hover:bg-neutral-200 dark:hover:bg-neutral-700`
      - ghost: `text-neutral-600 dark:text-neutral-400 hover:bg-neutral-100 dark:hover:bg-neutral-800`
      - outline: `border-2 border-primary-500 text-primary-500 hover:bg-primary-500 hover:text-white`
    - Sizing: sm (`px-4 py-2 text-sm`), md (`px-6 py-3 text-base`), lg (`px-8 py-4 text-lg`)
    - Shared: `rounded-xl font-semibold transition-all duration-300 ease-spring hover:scale-105 active:scale-95 focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-primary-500 focus-visible:ring-offset-2`
    - Use Svelte 5 `$props()` and `Snippet` type for children
    - Follow the exact pattern from RESEARCH.md Pattern 5

    **Card component** (`src/lib/components/ui/card.svelte`):
    - Props: `variant` ('default' | 'highlighted'), `hover` (boolean, default true), `class` (optional), `children` (Snippet)
    - Default: `bg-white dark:bg-neutral-900 border border-neutral-200 dark:border-neutral-800 rounded-2xl`
    - Highlighted: `bg-primary-500 text-white shadow-glow-lg` (for pricing featured tier)
    - Hover effects (when hover=true): `hover:shadow-xl hover:-translate-y-1 hover:border-primary-300 dark:hover:border-primary-700`
    - Inner content wrapper with `group-hover:scale-[1.02]` transform
    - Transition: `transition-all duration-300 ease-out`

    **Badge component** (`src/lib/components/ui/badge.svelte`):
    - Props: `variant` ('default' | 'accent' | 'outline' | 'success'), `size` ('sm' | 'md'), `children` (Snippet)
    - Default: `bg-primary-100 dark:bg-primary-900 text-primary-700 dark:text-primary-300`
    - Accent: `bg-accent-500 text-white`
    - Outline: `border border-neutral-300 dark:border-neutral-700 text-neutral-600 dark:text-neutral-400`
    - Success: `bg-success/10 text-success`
    - Sizing: sm (`px-2 py-0.5 text-xs`), md (`px-3 py-1 text-sm`)
    - Shared: `inline-flex items-center rounded-full font-semibold`

    **Section component** (`src/lib/components/ui/section.svelte`):
    - Props: `id` (optional), `variant` ('light' | 'dark' | 'gradient'), `padding` ('sm' | 'md' | 'lg', default 'lg'), `class` (optional), `children` (Snippet)
    - Light: `bg-white dark:bg-neutral-950`
    - Dark: `bg-neutral-950 dark:bg-neutral-900 text-white`
    - Gradient: `bg-gradient-to-br from-primary-950 via-neutral-950 to-primary-900 text-white`
    - Padding: sm (`py-12 md:py-16`), md (`py-16 md:py-24`), lg (`py-24 md:py-32`)
    - Renders as `<section>` element with the id prop

    **Container component** (`src/lib/components/ui/container.svelte`):
    - Props: `size` ('sm' | 'md' | 'lg' | 'xl', default 'lg'), `class` (optional), `children` (Snippet)
    - Sizing: sm (`max-w-3xl`), md (`max-w-5xl`), lg (`max-w-6xl`), xl (`max-w-7xl`)
    - Shared: `mx-auto px-4 sm:px-6 lg:px-8`

    **ThemeToggle component** (`src/lib/components/ui/theme-toggle.svelte`):
    - Import `toggleMode` and `mode` from `mode-watcher`
    - Import `Sun` and `Moon` from `lucide-svelte`
    - Renders a button that toggles between light/dark
    - Shows Sun icon in dark mode, Moon icon in light mode
    - Styling: `p-2 rounded-lg transition-all duration-200 hover:bg-neutral-100 dark:hover:bg-neutral-800`
    - aria-label for accessibility (use Paraglide `m.ui_theme_toggle()`)

    **LanguageSwitcher component** (`src/lib/components/ui/language-switcher.svelte`):

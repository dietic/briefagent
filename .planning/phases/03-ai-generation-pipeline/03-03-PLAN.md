       import { z } from 'zod';

       export const brandAnalysisSchema = z.object({
         primaryColors: z.array(z.string()).describe('Hex color codes of dominant brand colors'),
         secondaryColors: z.array(z.string()).describe('Hex color codes of accent/secondary colors'),
         visualStyle: z.string().describe('Overall visual style: minimalist, bold, playful, corporate, etc.'),

       import { z } from 'zod';

         body: z.string().describe('Value-driven body content, professional but human'),
         cta: z.string().describe('Goal-aligned call to action'),
         hashtags: z.array(z.string()).min(3).max(5).describe('Mix of popular and niche hashtags, without # prefix'),

    6. Create `src/lib/server/ai/pipeline/brand-analyzer.ts`:
       - Import `generateText, Output` from 'ai', `analysisModel` from '../providers', `brandAnalysisSchema, BrandAnalysis` from schemas, `buildBrandAnalysisPrompt` from prompts
       - Import `db` from '$lib/server/db', `products` from schema, `eq` from 'drizzle-orm'

       Export `async function analyzeBrandAssets(assetUrls: string[]): Promise<BrandAnalysis>`:
       - Take up to 5 asset URLs (slice if more)
       - Build image content array: `assetUrls.map(url => ({ type: 'image' as const, image: url }))`
       - Call `generateText` with `model: analysisModel`, `output: Output.object({ schema: brandAnalysisSchema })`, messages array with system prompt + user message containing text + images
       - Return the typed BrandAnalysis output

       Export `async function getCachedBrandAnalysis(productId: string, assetUrls: string[]): Promise<BrandAnalysis>`:
       - Query product: `db.query.products.findFirst({ where: eq(products.id, productId) })`
       - If product.brandAnalysis exists (not null), parse and return it as BrandAnalysis
       - Otherwise, call `analyzeBrandAssets(assetUrls)`
       - Save result: `db.update(products).set({ brandAnalysis: result }).where(eq(products.id, productId))`
       - Return result

    7. Add `brandAnalysis` column to products table in `src/lib/server/db/schema.ts`:
       - Add `brandAnalysis: jsonb('brand_analysis')` to the products table definition
       - This caches the brand analysis result per product (Pitfall 4 from research: avoid re-running vision on every image generation)

    8. Create `src/lib/server/ai/pipeline/copy-generator.ts`:
       - Import `generateText, Output` from 'ai', `copyModel` from '../providers', `copyOutputSchema, CopyOutput` from schemas, `buildCopySystemPrompt, buildCopyUserPrompt` from prompts, `AssembledBrief` type

       Export `async function generatePostCopy(postSlot: { topic: string; contentCategory: string; keyMessage: string }, brief: AssembledBrief): Promise<CopyOutput>`:
       - Call `generateText` with model: copyModel, output: Output.object({ schema: copyOutputSchema }), system and prompt from builders
       - Return typed CopyOutput

    9. Create `src/lib/server/ai/pipeline/image-generator.ts`:
       - Import `generateImage` from 'ai', `imageModel` from '../providers', `sharp` from 'sharp', `BrandAnalysis` type, `buildImagePrompt` from '../prompts/image-system'
       - Import `supabaseAdmin` from '$lib/server/supabase-admin'

       Export `async function generatePostImage(postTopic: string, keyMessage: string, brandAnalysis: BrandAnalysis, productId: string, postId: string): Promise<{ imageUrl: string; imagePrompt: string }>`:
       - Build prompt via `buildImagePrompt(postTopic, keyMessage, brandAnalysis)`
       - Call `generateImage` with model: imageModel, prompt, size: '1024x1024', providerOptions: { openai: { quality: 'medium' } }
       - Convert result to Buffer: `Buffer.from(image.uint8Array)`
       - Resize with Sharp to 1200x1200: `sharp(buffer).resize(1200, 1200, { fit: 'cover' }).jpeg({ quality: 90 }).toBuffer()`
       - Upload to Supabase Storage bucket 'generated-images' at path `{productId}/{postId}.jpg`
         ```
         await supabaseAdmin.storage.from('generated-images').upload(filePath, resized, {
           contentType: 'image/jpeg', upsert: true
         });
         const { data: { publicUrl } } = supabaseAdmin.storage.from('generated-images').getPublicUrl(filePath);
         ```
       - Return `{ imageUrl: publicUrl, imagePrompt: prompt }`

       NOTE: Use 'generated-images' as a SEPARATE bucket from user 'assets' bucket. This keeps generated content distinct from user uploads.

       Export `async function generatePostsForPlan(jobId: string, contentPlanId: string, productId: string): Promise<void>`:
       - Helper: `updateJob(jobId, updates)` -- same pattern as plan generation

       Steps:
       a) Update job: status 'running', step 1, progress 'Assembling brief...'
          Assemble brief: `assembleBrief(productId)`

       b) Update job: step 2, progress 'Analyzing brand assets...'
          Get asset URLs from brief.assets (filter for image types, map to fileUrl)
          If assets exist: `getCachedBrandAnalysis(productId, assetUrls)`
          If no assets: use a default BrandAnalysis with generic professional values

       c) Query all posts for this content plan: `db.query.posts.findMany({ where: eq(posts.contentPlanId, contentPlanId) })`
          Set totalSteps = 2 + (postCount * 2) -- 1 brief + 1 brand analysis + N copies + N images

       d) Generate copy for each post SEQUENTIALLY with batch of 3 concurrent calls:
          - Process posts in batches of 3 using Promise.all
          - For each post: `generatePostCopy({ topic, contentCategory, keyMessage }, brief)`
          - Update post row: `db.update(posts).set({ copyText: copy.fullText, hashtags: copy.hashtags, updatedAt: new Date() }).where(eq(posts.id, post.id))`
          - Update job progress after each batch: 'Generating copy {n}/{total}...'
          - Add 200ms delay between batches to respect rate limits

       e) Generate images for each post (only for static_image postType) SEQUENTIALLY:
          - Process ONE image at a time (image generation is slower, avoid rate limits)
          - For each static_image post: `generatePostImage(post.topic, post.keyMessage, brandAnalysis, productId, post.id)`
          - Update post row: `db.update(posts).set({ imageUrl: result.imageUrl, imagePrompt: result.imagePrompt, updatedAt: new Date() }).where(eq(posts.id, post.id))`
          - Update job progress: 'Generating image {n}/{imageTotal}...'
          - Add 500ms delay between image calls

       f) Update all posts status from 'draft' to 'pending_review'
       g) Update job: status 'completed', progress 'All posts generated!'

       Wrap everything in try/catch -- on error, update job status 'failed' with error message.

  <files>

  <action>
1. In `src/lib/server/db/schema.ts`, add a `platform` column to the `contentPillars` table:
   ```typescript
   platform: text('platform'),  // nullable: 'linkedin' | 'x' | null
   ```
   Add it after the `description` column, before `sortOrder`. This follows prior decision [06-01]: plain text column, not DB enum.

2. Run `pnpm generate` then `pnpm migrate` to create and apply the migration. If the migration system has issues (like it did before), fall back to `pnpm drizzle-kit push` as a last resort.

3. Create `src/lib/server/ai/platform-specs.ts` with the shared `PLATFORM_SPECS` config object. This is the single source of truth for all platform-specific behavior across the entire pipeline.

   Export:
   - `PlatformSpec` interface with: slug, displayName, charLimit, charRecommended ({min, max}), imageSize ({width, height}), imageAspectRatio, hashtagRules, toneGuidelines, active
   - `PLATFORM_SPECS: Record<string, PlatformSpec>` with entries for linkedin, x, instagram, youtube, tiktok (values from research file)
   - `ACTIVE_PLATFORMS` = filter where active === true (linkedin, x)
   - `COMING_SOON_PLATFORMS` = filter where active === false (instagram, youtube, tiktok)
   - `getPlatformSpec(slug: string | null): PlatformSpec` — returns PLATFORM_SPECS[slug] or falls back to PLATFORM_SPECS.linkedin as default

   Platform specs (from verified research):
   - LinkedIn: 3000 chars, recommended 1200-1800, image 1200x1200, 1:1, "3-5 hashtags: mix of 2 popular/broad + 2-3 niche/specific", professional but human tone, active: true
   - X: 280 chars, recommended 200-280, image 1200x675, 16:9, "1-2 hashtags maximum, integrated naturally into text", concise/punchy/opinionated tone, active: true
   - Instagram: 2200 chars, recommended 150-500, image 1080x1350, 4:5, active: false
   - YouTube: 5000 chars, recommended 200-500, image 1280x720, 16:9, active: false
   - TikTok: 2200 chars, recommended 50-150, image 1080x1920, 9:16, active: false
  </action>
  <verify>
    - `pnpm build` succeeds (no TypeScript errors in new file)
    - `platform-specs.ts` exports all 5 items
    - `ACTIVE_PLATFORMS` has exactly 2 entries (linkedin, x)
    - `COMING_SOON_PLATFORMS` has exactly 3 entries
    - `getPlatformSpec(null)` returns linkedin spec
    - `getPlatformSpec('x')` returns x spec
  </verify>
  <done>
    content_pillars table has nullable platform text column. PLATFORM_SPECS config exists with all 5 platforms, active/coming-soon separation, and getPlatformSpec helper that defaults to LinkedIn.
  </done>

     ```typescript
     let pillars = $state<Array<{ name: string; description: string; platform: string }>>(

     );
     ```

   - Update `addSuggestion()` to include `platform: ''` in the new pillar object

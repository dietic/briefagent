  <action>
    **1. Extend schema (src/lib/server/db/schema.ts):**
    - Add `'published'` to `postStatusEnum` (after `'scheduled'`)
    - Add `rejectionReason: text('rejection_reason')` to `posts` table
    - Add `publishedAt: timestamp('published_at', { withTimezone: true })` to `posts` table
    - No other schema changes needed

    **2. Create shared status utility (src/lib/utils/post-status.ts):**
    - Export `PostStatus` type: `'draft' | 'pending_review' | 'approved' | 'rejected' | 'scheduled' | 'published'`
    - Export `postStatusColors` record mapping each status to `{ color: string; bg: string; label: string }`:
      - draft: `{ color: 'var(--text-muted)', bg: 'rgba(156,163,175,0.1)', label: 'Draft' }`
      - pending_review: `{ color: 'var(--c-secondary)', bg: 'rgba(249,115,22,0.1)', label: 'Review' }`
      - approved: `{ color: 'var(--c-electric)', bg: 'rgba(6,182,212,0.1)', label: 'Approved' }`
      - scheduled: `{ color: '#22c55e', bg: 'rgba(34,197,94,0.1)', label: 'Scheduled' }`
      - published: `{ color: '#16a34a', bg: 'rgba(22,163,74,0.1)', label: 'Published' }`
      - rejected: `{ color: 'var(--negative)', bg: 'rgba(239,68,68,0.1)', label: 'Rejected' }`

    **3. Create PATCH endpoint (src/routes/api/posts/[id]/+server.ts):**
    - Import `db`, `posts`, `products` from schema, `eq` from drizzle-orm, `z` from zod
    - Zod `updateSchema`:
      - `status`: optional enum of all 6 statuses
      - `copyText`: optional string
      - `hashtags`: optional array of strings
      - `scheduledAt`: optional string (ISO datetime)
      - `rejectionReason`: optional string
      - Refine: if `status === 'rejected'`, `rejectionReason` must be present
    - `PATCH` handler:
      1. `locals.getUser()` — 401 if not authenticated
      2. Parse + validate body with Zod — 400 on failure
      3. Load post with `db.query.posts.findFirst` using `with: { product: { columns: { userId: true } } }`
      4. Verify `post.product.userId === user.id` — 404 if not owned
      5. Build update object: set each field if present in parsed data, always set `updatedAt: new Date()`
      6. If `status === 'published'`, also set `publishedAt: new Date()`
      7. If `scheduledAt` present, convert string to `new Date()`
      8. `db.update(posts).set(updates).where(eq(posts.id, params.id))`
      9. Return `json({ success: true })`

    **4. Create regenerate endpoint (src/routes/api/posts/[id]/regenerate/+server.ts):**
    - Zod schema: `{ type: z.enum(['copy', 'image', 'both']) }`
    - `POST` handler:
      1. Auth check — 401
      2. Parse body — 400
      3. Load post `with: { product: true }` — verify ownership — 404
      4. `assembleBrief(post.productId)` to get brief context
      5. If type is `'copy'` or `'both'`: call `generatePostCopy({ topic: post.topic, contentCategory: post.contentCategory, keyMessage: post.keyMessage }, brief)`, set `updates.copyText = copy.fullText`, `updates.hashtags = copy.hashtags`
      6. If type is `'image'` or `'both'`: load brand analysis via `getCachedBrandAnalysis(post.productId, imageAssets)` where `imageAssets = brief.assets.filter(a => a.tag !== 'testimonial').map(a => a.fileUrl)`. If no assets, use default fallback object (same pattern as post-generator.ts). Call `generatePostImage(post.topic, post.keyMessage, brandAnalysis, post.productId, post.id)`, set `updates.imageUrl = result.imageUrl`, `updates.imagePrompt = result.imagePrompt`
      7. Set `updates.status = 'pending_review'` (reset after regen) and `updates.updatedAt = new Date()`
      8. `db.update(posts).set(updates).where(eq(posts.id, params.id))`
      9. Return `json({ success: true })`
  </action>

    **1. Create calendar server load (src/routes/dashboard/calendar/+page.server.ts):**
    - Import `db`, `posts` from schema, `eq`, `and`, `gte`, `lte` from drizzle-orm
    - Import `startOfMonth`, `endOfMonth`, `addMonths`, `subMonths`, `format` from date-fns
    - `load` function:
      1. Get `product` from `await parent()` — return `{ posts: [], year: now.getFullYear(), month: now.getMonth() }` if no product
      2. Read `month` param from `url.searchParams.get('month')` — if present, parse as `YYYY-MM` format to get year/month. If absent, use current date.
      3. Compute `monthStart = startOfMonth(targetDate)` and `monthEnd = endOfMonth(targetDate)`
      4. Query: `db.query.posts.findMany({ where: and(eq(posts.productId, product.id), gte(posts.scheduledAt, monthStart), lte(posts.scheduledAt, monthEnd)), columns: { id, topic, status, scheduledAt, platform, postType, imageUrl, contentCategory, copyText, hashtags }, orderBy: posts.scheduledAt })`
      5. Return `{ posts: mapped (scheduledAt as ISO string), year, month (0-indexed) }` for the calendar grid

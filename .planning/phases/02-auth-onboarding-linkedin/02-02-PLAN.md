    2. Create `src/lib/server/db/schema.ts` with all Phase 2 tables. Follow the EXACT schema from 02-RESEARCH.md Pattern 2:
       - `products`: id (uuid pk), userId (uuid, not null), name (text), websiteUrl (text), description (text), logoUrl (text), scrapedData (jsonb), onboardingStep (text, default 'quick_start'), createdAt, updatedAt
       - `productBriefs`: id (uuid pk), productId (uuid, unique ref to products with cascade delete), problemSolved, keyFeatures (text array), differentiator, pricingInfo, productStage, idealCustomer, industry, ageRange, painPoints (text array), audienceHangouts (text array), personalityTraits (text array), exampleContent, wordsToUse (text array), wordsToAvoid (text array), mainGoal, postingFrequency, preferredTimes (jsonb), createdAt, updatedAt
       - `assets`: id (uuid pk), productId (uuid ref to products with cascade delete), fileUrl, fileName, fileType, fileSize (integer), tag (text -- screenshot | photo | logo | lifestyle | testimonial | graphic), description, isPrimary (boolean default false), createdAt
       - `platformAccounts`: id (uuid pk), productId (uuid ref to products with cascade delete), platform (text), accountType (text), platformUserId (text), platformUserName (text), platformAvatarUrl (text), accessToken (text -- encrypted), refreshToken (text -- encrypted), tokenExpiresAt (timestamp), refreshTokenExpiresAt (timestamp), status (text default 'active'), createdAt, updatedAt

       Use `uuid('id').primaryKey().defaultRandom()` for all primary keys.
       Use `timestamp('created_at', { withTimezone: true }).defaultNow()` for timestamps.

    3. Create `src/lib/server/db/index.ts`:
       ```typescript
       import { drizzle } from 'drizzle-orm/postgres-js';
       import postgres from 'postgres';
       import { SUPABASE_DB_URL } from '$env/static/private';
       import * as schema from './schema';
       const client = postgres(SUPABASE_DB_URL, { prepare: false });
       export const db = drizzle(client, { schema });
       ```
       `prepare: false` is required for Supabase's connection pooler (PgBouncer in transaction mode).

    4. Create `src/lib/server/scraping/extractors.ts` with helper functions:
       - `extractKeyPhrases($: CheerioAPI): string[]` -- extracts from meta keywords or h1/h2 headings (max 10)
       - `extractColors($: CheerioAPI): string[]` -- extracts from theme-color meta, CSS custom properties in inline styles (best effort, max 5)

    5. Create `src/lib/server/scraping/index.ts` with `scrapeUrl(url: string)`:
       - 10-second timeout via AbortController
       - Fetches URL with a browser-like User-Agent
       - Parses with cheerio
       - Extracts: title (og:title fallback to <title>), description (og:description fallback to meta description), ogImage, themeColor, favicon, keyPhrases
       - Returns `ScrapedData` object with `success: boolean` and `error?: string` for graceful fallback
       - Follow the EXACT pattern from 02-RESEARCH.md Pattern 5

    6. Create `src/lib/server/storage.ts` with helpers:
       - `uploadAssetToStorage(supabase, file: File, userId: string, productId: string)`: Uploads to Supabase Storage `assets` bucket at path `{userId}/{productId}/{uuid}-{filename}`, returns public URL
       - Constants: `ALLOWED_IMAGE_TYPES = ['image/jpeg', 'image/png', 'image/gif', 'image/webp']`, `MAX_FILE_SIZE = 10 * 1024 * 1024` (10MB)
       - Validation function: `validateUpload(file: File): { valid: boolean, error?: string }`

    7. Create `src/routes/api/scrape/+server.ts`:
       - POST handler: accepts `{ url: string }` JSON body
       - Requires auth (call `locals.getUser()`, 401 if null)
       - Validates URL format (try `new URL(url)`)
       - Calls `scrapeUrl(url)` and returns the result as JSON
       - Always returns 200 even on scrape failure (the ScrapedData.success field indicates outcome)

    8. Create `src/routes/api/assets/upload/+server.ts`:
       - POST handler: accepts multipart FormData with `file`, `product_id`, `tag` (optional), `description` (optional)
       - Requires auth
       - Validates file type and size using `validateUpload`
       - Uploads via `uploadAssetToStorage`
       - Inserts asset record into DB via Drizzle
       - Returns the created asset as JSON
       - Follow the EXACT pattern from 02-RESEARCH.md Pattern 6

    9. After creating all files, push the schema to Supabase:
       ```bash
       npx drizzle-kit push
       ```
       This creates the tables directly (no migration files needed for dev). If the user hasn't set SUPABASE_DB_URL yet, this will fail -- that's expected and will be done during the checkpoint.

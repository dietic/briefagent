    **Create post-review-dialog.svelte using Bits UI Dialog compound component:**

    **Props:**
    - `post`: `{ id: string; topic: string; copyText: string | null; hashtags: string[] | null; imageUrl: string | null; scheduledAt: string | null; status: string; platform: string; keyMessage: string; contentCategory: string }` (matches server-loaded post shape)
    - `open`: `$bindable(false)` boolean
    - `onupdate`: optional callback `() => void` to trigger `invalidateAll()` from the parent

    **Layout (Bits UI Dialog.Root > Dialog.Portal > Dialog.Overlay + Dialog.Content):**
    - Overlay: fixed inset-0, bg-black/50, z-40
    - Content: fixed, centered, max-w-4xl, max-h-[90vh], rounded-2xl, overflow-y-auto, bg var(--bg-surface), border var(--border-subtle), z-50
    - Dialog.Title: post topic as heading
    - Dialog.Close: X button top-right

    **Content sections inside dialog:**

    1. **Image preview (top):**
       - If `post.imageUrl`: show image in rounded container, max-h-64, object-cover
       - Below image: two buttons side by side:
         - "Download Image" button (Download icon from lucide-svelte): calls `downloadImage()` helper function
         - "Regenerate Image" button (RefreshCw icon): calls regenerate API with `type: 'image'`, shows spinner while loading
       - `downloadImage` helper: `fetch(post.imageUrl)` -> `.blob()` -> `URL.createObjectURL()` -> create `<a>` with `.download` attr, click, revoke URL. Filename: `briefagent-{post.id.slice(0,8)}.jpg`
       - If no imageUrl: show placeholder div with Image icon

    2. **Copy editing (middle):**
       - Bind textarea to `$state` variable `editedCopy` initialized from `post.copyText ?? ''`
       - Character count below textarea: `$derived` value `editedCopy.length` displayed as `"{count} characters"` with color hint if over 3000 (LinkedIn limit)
       - "Copy to Clipboard" button (Copy icon): calls `navigator.clipboard.writeText(editedCopy)`, shows transient "Copied!" feedback for 2 seconds via `$state` boolean

    3. **Hashtag editing:**
       - `$state` array `editedHashtags` initialized from `post.hashtags ?? []`
       - Display as chips with X remove button on each
       - Text input below: on Enter keydown, add trimmed value (with `#` prefix if missing) to array, clear input
       - Follow the chip pattern from onboarding (Phase 2): `$state` array + Enter handler + X remove

    4. **Scheduled date editing:**
       - `$state` string `editedScheduledAt` initialized from `post.scheduledAt ?? ''`
       - `<input type="datetime-local">` bound to the state value
       - Format the initial value for the datetime-local input (strip timezone, format as `YYYY-MM-DDTHH:mm`)

    5. **Action buttons (bottom, flexbox row with gap):**
       - **"Approve" button** (Check icon, green bg): PATCH with `{ status: 'scheduled', copyText: editedCopy, hashtags: editedHashtags, scheduledAt: editedScheduledAt || undefined }` — this is the "edit and approve in one action" (REVW-11). Sets status to 'scheduled' per REVW-06. Calls `invalidateAll()` via `onupdate` callback after success, closes dialog.
       - **"Reject" button** (Ban icon, red bg): shows a textarea for rejection reason (`$state` string). Only enabled when reason is non-empty. PATCH with `{ status: 'rejected', rejectionReason }`. Calls onupdate + closes dialog.
       - **"Regenerate Copy" button** (RefreshCw icon): POST to regenerate with `{ type: 'copy' }`. Shows spinner. On success, refetch post data or update local state.
       - **"Regenerate Both" button**: POST to regenerate with `{ type: 'both' }`. Shows spinner.
       - **"Mark Published" button** (only visible when status is 'scheduled'): PATCH with `{ status: 'published' }`.
       - All buttons show loading state (`$state` booleans) while API calls are in flight. Disable buttons during loading.

    **Helper functions inside the component:**
    - `async function updatePost(body: Record<string, unknown>)`: calls PATCH `/api/posts/${post.id}`, calls `onupdate?.()`, closes dialog
    - `async function regeneratePost(type: 'copy' | 'image' | 'both')`: calls POST `/api/posts/${post.id}/regenerate`, calls `onupdate?.()`. Set loading state per regeneration type.
    - `async function copyToClipboard()`: `navigator.clipboard.writeText(editedCopy)` with fallback (execCommand), set copied toast state
    - `async function downloadImage()`: fetch blob + create download link pattern from research

Output: Updated brief assembler that fetches pillars, updated AI prompts that include pillar context and distribution instructions.
</objective>

    2. In `buildPlanUserPrompt()`, after the "Key Features & differentiator" section (around line 31-42) and before the "Target audience" section, add a conditional Content Pillars section:
       ```
       // Content Pillars (personal brand)
       if (brief.contentPillars.length > 0) {
         const pillarLines = brief.contentPillars.map(
           (p, i) => `${i + 1}. **${p.name}**${p.description ? `: ${p.description}` : ''}`
         );
         sections.push(`## Content Pillars
    This is a personal brand. Distribute posts across these content pillars:
    ${pillarLines.join('\n')}

    Each pillar should receive roughly equal coverage. Tag each post's topic to align with one of these pillars.`);
       }
       ```

    3. When content pillars are present, SKIP the product-specific sections (Key Features, Differentiator, Problem Solved) since personal brands don't have those fields. Wrap the existing key features / differentiator / problemSolved sections in a condition:
       ```
       if (brief.contentPillars.length === 0) {
         // existing key features, differentiator, problemSolved sections
       }
       ```
       This ensures product/service types get the original prompt structure, while personal brands get pillar context instead.
